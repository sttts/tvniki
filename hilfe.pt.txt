.topic Installation
 █ Instalação
 █████████████

Para instalar o tvNiki no seu computador, copie-o para uma pasta à sua escolha.
A versão mínima do tvNiki consiste no ficheiro NIKI.EXE. Para usar a ajuda,
também é necessário o ficheiro HILFE.HLP.

Para iniciar o Niki, basta digitar NIKI se estiver na pasta do Niki. Para
poder iniciar o Niki a partir de outras pastas, pode expandir o caminho de
pesquisa no AUTOEXEC.BAT para que o DOS pesquise automaticamente na pasta do
Niki, ou criar um ficheiro batch simples que chame o tvNiki.

O tvNiki precisa de permissões de escrita na pasta atual. Isto significa que
o uso em rede só pode ocorrer a partir de uma unidade que não esteja protegida
contra escrita. O importante é que apenas a pasta atual precisa ter esta
propriedade. Não importa se a pasta que contém o NIKI.EXE é gravável ou não.

É suficiente disponibilizar apenas uma versão do NIKI.EXE na rede para todos
os utilizadores.

O uso do tvNiki em rede NÃO se enquadra no uso privado, portanto uma licença
deve ser adquirida junto do {autor:Copyright}.

.topic Hardware
 █ Requisitos do sistema
 ████████████████████████

Para usar o Niki, o seu computador deve ter pelo menos 400KB de memória DOS
livre. Todos os processadores a partir do i8088 são suportados. São necessários
pelo menos 100KB no disco rígido ou disquete. Além disso, deve ser usado pelo
menos o MS-DOS 3.3.

O tvNiki é totalmente compatível com o Windows 95. No entanto, nomes de
ficheiros longos não podem ser usados.

O uso sob Windows NT ainda não foi testado. Portanto, erros não podem ser
excluídos.

Numa {rede:Installation}, o Niki pode ser disponibilizado para todos os
utilizadores. No entanto, o bloqueio real de ficheiros ainda não é suportado.
Mesmo assim, não há problemas em instalar o tvNiki num servidor Netware,
por exemplo.

.topic Teach
 █ Teach-In
 ███████████

O tvNiki possui um modo Teach-In. Normalmente, um programa deve ser inserido
linha por linha através do {editor:Editor}. Para tarefas curtas e simples, no
entanto, este modo também está disponível, no qual o Niki pode ser controlado
com várias teclas. A {sequência de comandos:Befehle} é armazenada na área de
transferência e pode ser guardada como um programa completo executável no
final da gravação.

Para entrar no modo Teach-In, selecione o item de menu Compilador|Teach-In.
Agora pode rodar o Niki e avançar com as teclas de seta Esquerda e Cima. Com
a tecla Delete, o Niki apanha paletes. Com Insert, ele deposita-as.

Se ocorrer um erro ou se interromper a gravação com ESC, aparece uma caixa de
diálogo perguntando se deseja guardar o programa. Quer o guarde ou não, ele
está agora na área de transferência e pode ser {colado:Blockbefehle} para
continuar a trabalhar.

.topic Feldeditor
 █ Editor de campo
 ██████████████████

O editor de campo consiste numa janela preta que contém uma divisão de 15 por
10 campos. Nela pode criar campos de robô nos quais o Niki pode andar.

Para colocar obstáculos pelos quais o Niki não pode passar, use o rato clicando
com o botão esquerdo entre dois pontos, ou mova o cursor com as teclas de seta
e coloque obstáculos com a barra de espaço.

Clicando novamente ou premindo Espaço novamente, as paredes são removidas.

Para mover o Niki, clique com o botão direito do rato no centro entre 4 pontos.
Clicar novamente roda o Niki sobre o seu próprio eixo. Com o teclado, isto é
feito com a tecla P.

Com as teclas + e -, pode depositar ou remover paletes na posição atual do
cursor. Para mover o cursor, clique com o botão esquerdo do rato na posição
desejada ou mova-o com as teclas de seta.

Para dar ao Niki uma reserva de paletes antes de iniciar um programa,
selecione o item de menu Campo|Reserva.

A velocidade de execução de um programa é definida através do item de menu
Campo|Velocidade.

.topic   Kommentare
 █ Comentários
 ██████████████

Comentários são descrições e complementos aos textos de programa que NÃO são
processados pelo compilador, ou seja, não têm absolutamente nenhum significado
para o funcionamento do programa, mas tornam um programa mais legível e
compreensível se usados corretamente.

Em PASCAL existem dois tipos de comentários. Um é delimitado por chavetas, o
outro por parênteses seguidos de um asterisco:

  {Isto é um comentário}
  (* Isto também é um comentário *)

Comentários podem estar em qualquer lugar no programa, mesmo dentro de
instruções {IF-THEN:If}, etc.

  IF Vorne_Frei {Aqui o comentário} THEN Vor;

.topic Sensoren
 █ Sensores
 ███████████

O tvNiki fornece vários sensores que não fazem parte do padrão PASCAL.
No entanto, o uso é exatamente igual ao das funções PASCAL.

Todos os sensores do tvNiki têm um valor de retorno Booleano, ou seja, têm
ou o valor TRUE (verdadeiro) ou o valor FALSE (falso). Nenhuma das funções
retorna um valor numérico. O Niki pode, por exemplo, apenas determinar se
ainda tem paletes carregadas, mas não o número de paletes carregadas.

Sensores só podem ser usados em {estruturas:Strukturen}, o uso como
{comandos:Befehle} não é possível.

 {Platz_Belegt}

 {Vorne_Frei}
 {Links_Frei}
 {Rechts_Frei}

 {Hat_Vorrat}

 Exemplos:

   IF {Vorne_Frei} THEN {Vor};

   WHILE {Hat_Vorrat} DO {Gib_Ab};

 IF {Sensor:Sensoren} THEN {Comando:Befehle};

Se o sensor entre IF e THEN tiver o valor TRUE, o comando ou bloco após THEN

sensores.

 Exemplo:

   IF Hat_Vorrat AND Vorne_Frei THEN

.topic While
 █ WHILE DO
 ███████████

 WHILE {Sensor:Sensoren} DO {Comando:Befehle};

 WHILE {Sensor:Sensoren} DO
 BEGIN
   {Comando:Befehle};

 Exemplo:

   WHILE Hat_Vorrat DO

   END;

sensores.

.topic Repeat
 █ REPEAT UNTIL
 ███████████████

 REPEAT {Comando:Befehle}; UNTIL {Sensor:Sensoren};

 REPEAT
   {Comando:Befehle};
   {Comando:Befehle};
 UNTIL {Sensor:Sensoren};

O comando ou bloco entre REPEAT e UNTIL é executado até que o sensor após
UNTIL retorne o valor TRUE.

 Exemplo:

   REPEAT {Vor}; UNTIL NOT {Vorne_Frei};


Aqui {REPEAT UNTIL:Repeat} executa primeiro o comando e depois verifica se o
sensor é TRUE. Em contraste, com {WHILE DO:While} primeiro verifica-se se o
sensor é TRUE, só depois o comando é executado.

sensores.

 Exemplo:

   REPEAT
     Gib_Ab;
     Vor;
   UNTIL NOT (Hat_Vorrat AND Vorne_Frei);

.topic Logik
 █ Operações lógicas
 ████████████████████

{Sensores:Sensoren} podem ser combinados em PASCAL com operadores lógicos.
Assim pode realizar consultas que dependem de mais de um sensor.

 Exemplo:

   WHILE (Hat_Vorrat AND Vorne_Frei) DO
   BEGIN
     Gib_Ab;
     Vor;
   END;

Para isso estão disponíveis os dois operadores OR e AND.


O operador OR combina dois {sensores:Sensoren} de forma que o valor de retorno
seja TRUE se pelo menos um dos dois retornar TRUE.

 Exemplo:

   IF Links_Frei OR Rechts_Frei THEN Vor;

 Tabela de verdade:

   Links_Frei OR Rechts_Frei =
   ████████████████████████████████████
   FALSE      │  FALSE       │ FALSE
   TRUE       │  FALSE       │ TRUE
   FALSE      │  TRUE        │ TRUE
   TRUE       │  TRUE        │ TRUE
   ████████████████████████████████████


O operador AND combina dois {sensores:Sensoren} de forma que o valor de retorno
seja TRUE se ambos os {sensores:Sensoren} retornarem TRUE.

 Exemplo:

   IF Links_Frei AND Rechts_Frei THEN Vor;

 Tabela de verdade:

   Links_Frei AND Rechts_Frei =
   ████████████████████████████████████
   FALSE      │  FALSE       │ FALSE
   TRUE       │  FALSE       │ FALSE
   FALSE      │  TRUE        │ FALSE
   TRUE       │  TRUE        │ TRUE
   ████████████████████████████████████


Usando mais de um operador, também pode combinar vários sensores.

 Exemplo:

   IF Links_Frei OR Rechts_Frei OR Vorne_Frei THEN Nimm_Auf;

Quando diferentes operadores aparecem em tal combinação, surge a questão da
ordem de avaliação. Em princípio aplica-se: NOT antes de AND, AND antes de OR.
Isto significa que primeiro as combinações NOT, depois AND e depois OR são
avaliadas.

 Exemplo:

   IF Vorne_Frei OR Links_Frei AND Rechts_Frei THEN ...

   corresponde a

   IF Vorne_Frei OR (Links_Frei AND Rechts_Frei) THEN ...


 Prioridade      corresponde em matemática
 ████████████████████████████████████████████
 1   │  NOT    │ - (negação)
 2   │  AND    │ */ (multiplicação/divisão)
 3   │  OR     │ +- (adição/subtração)
 ████████████████████████████████████████████

.topic Unterprogramme
 █ Subprogramas
 ███████████████

Em PASCAL pode substituir partes iguais num programa por subprogramas. Estes
subprogramas podem então ser usados exatamente como {comandos:Befehle} reais.

Estes subprogramas são chamados "procedimentos" em PASCAL. São definidos antes
do programa principal e são estruturados como pequenos programas PASCAL.

 Exemplo:

   PROGRAM Exemplo;

     PROCEDURE Drehe_Rechts;
     BEGIN
       Drehe_Links;
       Drehe_Links;
       Drehe_Links;
     END;

   BEGIN
     Vor;
     Drehe_Rechts;
   END.

Entre BEGIN e END podem ser usados quaisquer {comandos:Befehle} e
{estruturas:Strukturen}. Quando um procedimento é chamado, os comandos no
procedimento são executados um após o outro. Depois retorna-se ao ponto de
chamada.

O programa acima poderia portanto também ser escrito assim:

  PROGRAM Exemplo;
  BEGIN
    Vor;
    Drehe_Links;
    Drehe_Links;
    Drehe_Links;
  END;


No entanto, assim que um programa precisa virar à direita várias vezes, deve
usar um procedimento para isso, pois o programa torna-se assim mais curto e
sobretudo mais claro. Procedimentos fazem portanto parte de um bom estilo de
programação.


Procedimentos podem chamar a si próprios ou outros procedimentos, desde que
já estejam definidos no momento da chamada. Devem portanto estar definidos
acima da chamada no texto do programa. Quando procedimentos chamam a si
próprios, chama-se "programação recursiva". Na programação recursiva deve-se
ter cuidado para que não se forme um ciclo infinito, caso contrário ocorre
eventualmente um "stack overflow" e o programa é interrompido. Com programas
PASCAL reais, isto pode até causar um crash do computador.

 Exemplo:

   PROGRAM Errado;

     PROCEDURE Gehe;
     BEGIN
       Vor;
       Gehe;
     END;

   BEGIN
     Gehe;
   END.

Como a maioria dos problemas também pode ser resolvida sem esta técnica por
vezes confusa, a programação linear deveria ser preferida. A programação
recursiva só deveria realmente ser usada onde é absolutamente necessário.

.topic Aufbau
 █ Estrutura dos programas PASCAL
 █████████████████████████████████

Um programa PASCAL ou programa Niki tem sempre a mesma estrutura:

Na primeira linha é especificado um nome para o programa.

  PROGRAM Nome;

O nome não pode ser igual a uma palavra-chave PASCAL predefinida ou a um
comando do Niki. Um programa com o nome "Vor" não é portanto possível.

Depois vêm todas as definições. No tvNiki, estes são todos os
{procedimentos:Unterprogramme} definidos.

Por último, cada programa PASCAL deve conter um programa principal. Este
programa principal começa com BEGIN e termina com END seguido de um ponto.
Entre eles podem estar {comandos:Befehle}. Cada comando é terminado com um
ponto e vírgula.

  BEGIN
    {Drehe_Links};
    {Vor};
  END.

Ao iniciar o programa, a execução começa no primeiro comando do programa
principal.

Programas PASCAL válidos podem portanto ter este aspeto:

  PROGRAM Exemplo1;

  BEGIN
    {Vor};
    {Vor};
    {Drehe_Links};
  END.


  PROGRAM Exemplo2;

    {PROCEDURE:Unterprogramme} Armazenar;
    BEGIN
      {Drehe_Links};
      {Vor};
      {Gib_Ab};
      {Drehe_Links};
      {Drehe_Links};
      {Vor};
      {Drehe_Links};
    END;

  BEGIN
    {Nimm_Auf};
    {Nimm_Auf};
    {Nimm_Auf};
    Armazenar;
    Armazenar;
    Armazenar;
  END.
