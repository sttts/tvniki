.topic Installation
 █ Installation
 ███████████████

Pour installer tvNiki sur votre ordinateur, copiez-le dans un répertoire de
votre choix. La version minimale de tvNiki se compose du fichier NIKI.EXE.
Pour utiliser l'aide, le fichier HILFE.HLP est également nécessaire.

Pour démarrer Niki, tapez simplement NIKI si vous êtes dans le répertoire Niki.
Pour pouvoir démarrer Niki depuis d'autres répertoires, vous pouvez soit
étendre votre chemin de recherche dans AUTOEXEC.BAT pour que DOS recherche
automatiquement le répertoire Niki, soit créer un simple fichier batch qui
appelle tvNiki.

tvNiki nécessite des droits d'écriture dans le répertoire courant. Cela signifie
que l'utilisation en réseau ne peut se faire qu'à partir d'un lecteur non
protégé en écriture. L'important est que seul le répertoire courant ait besoin
de cette propriété. Peu importe si le répertoire contenant NIKI.EXE est
accessible en écriture ou non.

Il suffit de mettre à disposition une seule version de NIKI.EXE sur le réseau
pour tous les utilisateurs.

L'utilisation de tvNiki en réseau NE relève PAS de l'usage privé, une licence
doit donc être achetée auprès de l'{auteur:Copyright}.

.topic Hardware
 █ Configuration requise
 ████████████████████████

Pour utiliser Niki, votre ordinateur doit disposer d'au moins 400 Ko de mémoire
DOS libre. Tous les processeurs à partir du i8088 sont pris en charge. Au moins
100 Ko sont nécessaires sur le disque dur ou la disquette. De plus, MS-DOS 3.3
ou supérieur doit être utilisé.

tvNiki est entièrement compatible avec Windows 95. Cependant, les noms de
fichiers longs ne peuvent pas être utilisés.

L'utilisation sous Windows NT n'a pas encore été testée. Des erreurs ne peuvent
donc pas être exclues.

Dans un {réseau:Installation}, Niki peut être mis à disposition de tous les
utilisateurs. Cependant, le verrouillage réel des fichiers n'est pas encore
pris en charge. Néanmoins, il n'y a pas de problème pour installer tvNiki sur
un serveur Netware, par exemple.

.topic Teach
 █ Teach-In
 ███████████

tvNiki dispose d'un mode Teach-In. Normalement, un programme doit être saisi
ligne par ligne via l'{éditeur:Editor}. Cependant, pour des tâches courtes et
simples, ce mode est également disponible, dans lequel Niki peut être contrôlé
avec différentes touches. La {séquence de commandes:Befehle} est stockée dans
le presse-papiers et peut être sauvegardée comme programme complet exécutable
à la fin de l'enregistrement.

Pour accéder au mode Teach-In, sélectionnez l'élément de menu Compilateur|Teach-In.
Vous pouvez maintenant faire tourner Niki et avancer avec les touches fléchées
Gauche et Haut. Avec la touche Suppr, Niki ramasse des palettes. Avec Inser,
il en dépose.

Si une erreur se produit ou si vous interrompez l'enregistrement avec ESC, une
boîte de dialogue apparaît vous demandant si vous souhaitez sauvegarder le
programme. Que vous le sauvegardiez ou non, il se trouve maintenant dans le
presse-papiers et peut être {collé:Blockbefehle} pour continuer à travailler.

.topic Debug
 █ Mode pas à pas
 █████████████████

tvNiki dispose d'un mode pas à pas dans lequel un programme peut être exécuté
étape par étape. Il peut être utilisé pour rechercher des erreurs dans les
programmes ou pour vérifier des programmes.

Pour accéder à ce mode, appuyez soit sur F8 pendant l'exécution du programme,
soit démarrez le programme directement en mode pas à pas via Alt-F8.

En appuyant sur F9, un programme peut à nouveau être exécuté à pleine vitesse.

.topic Bedienung
 █ Utilisation de tvNiki
 ████████████████████████

Dans tvNiki, il y a trois fenêtres différentes. La fenêtre visible au démarrage
du programme est l'{éditeur de texte:Editor}, dans lequel vous pouvez saisir et
modifier des programmes.

La deuxième fenêtre est l'{éditeur de terrain:Feldeditor}. C'est là que les
terrains de robot, dans lesquels Niki le robot se déplace, sont modifiés.

La dernière fenêtre est la fenêtre Info. Dans la fenêtre Info, des informations
actuelles sur la {fenêtre de terrain:Feldeditor} sont constamment affichées.
Vous pouvez y lire par exemple la réserve actuelle et la position de Niki.
De plus, la ligne inférieure indique le mode actuel dans lequel se trouve
l'éditeur de terrain.

 {1. Utilisation de l'éditeur de texte:Editor}
 {2. Création et modification de terrains:Feldeditor}

.topic Feldeditor
 █ Éditeur de terrain
 █████████████████████

L'éditeur de terrain se compose d'une fenêtre noire contenant une grille de
15 par 10 cases. Vous pouvez y créer des terrains de robot dans lesquels Niki
peut se déplacer.

Pour placer des obstacles que Niki ne peut pas traverser, utilisez soit la
souris en cliquant avec le bouton gauche entre deux points, soit déplacez le
curseur avec les touches fléchées et placez des obstacles avec la barre d'espace.

En cliquant à nouveau ou en appuyant à nouveau sur Espace, les murs sont
supprimés.

Pour déplacer Niki, cliquez avec le bouton droit de la souris au centre de
4 points. Cliquer à nouveau fait tourner Niki sur lui-même. Avec le clavier,
cela se fait avec la touche P.

Avec les touches + et -, vous pouvez déposer ou retirer des palettes à la
position actuelle du curseur. Pour déplacer le curseur, cliquez avec le bouton
gauche de la souris à la position souhaitée ou déplacez-le avec les touches
fléchées.

Pour donner à Niki une réserve de palettes avant de démarrer un programme,
sélectionnez l'élément de menu Terrain|Réserve.

La vitesse d'exécution d'un programme est réglée via l'élément de menu
Terrain|Vitesse.

.topic Editor
 █ Utilisation de l'éditeur de texte
 ████████████████████████████████████

L'éditeur de tvNiki est un éditeur de texte normal compatible Wordstar. Il
permet de modifier des fichiers texte jusqu'à 64 Ko. Les fonctions de l'éditeur
peuvent être appelées via les menus Fichier, Édition et Recherche.

De plus, le texte dans l'éditeur, s'il s'agit d'un {programme PASCAL:PASCAL},
peut être compilé et exécuté dans l'{éditeur de terrain:Feldeditor}. Ces
fonctions peuvent être activées via le menu Compilateur. Si des modifications
ont été apportées au programme, une boîte de dialogue apparaît informant de
l'état actuel de la compilation. Si une erreur se produit, une ligne s'affiche
dans l'éditeur décrivant l'{erreur:Fehler}. Avec la touche F1, vous pouvez
afficher plus d'informations sur l'erreur survenue. De plus, l'éditeur saute
à la ligne où l'{erreur:Fehler} a été trouvée. Si aucune erreur ne s'est
produite, un bouton OK apparaît dans la fenêtre d'état, qui ferme la boîte
de dialogue en appuyant sur Entrée.

.topic   Kommentare
 █ Commentaires
 ███████████████

Les commentaires sont des descriptions et des compléments aux textes de
programme qui ne sont PAS traités par le compilateur, c'est-à-dire qu'ils
n'ont absolument aucune signification pour le fonctionnement du programme,
mais le rendent plus lisible et compréhensible s'ils sont utilisés correctement.

En PASCAL, il existe deux types de commentaires. L'un est entouré d'accolades,
l'autre par des parenthèses suivies d'une étoile:

  {Ceci est un commentaire}
  (* Ceci est également un commentaire *)

Les commentaires peuvent être placés n'importe où dans le programme, même à
l'intérieur d'une instruction {IF-THEN:If}, etc.

  IF Vorne_Frei {Voici le commentaire} THEN Vor;

.topic Sensoren
 █ Capteurs
 ███████████

tvNiki fournit divers capteurs qui ne font pas partie du standard PASCAL.
Cependant, leur utilisation est exactement la même que celle des fonctions
PASCAL.

Tous les capteurs tvNiki ont une valeur de retour booléenne, c'est-à-dire
qu'ils ont soit la valeur TRUE (vrai), soit la valeur FALSE (faux). Aucune des
fonctions ne renvoie de valeur numérique. Niki peut par exemple seulement
déterminer s'il a encore des palettes chargées, mais pas le nombre de palettes
chargées.

Les capteurs ne peuvent être utilisés que dans des {structures:Strukturen},
leur utilisation comme {commandes:Befehle} n'est pas possible.

 {Platz_Belegt}

 {Vorne_Frei}
 {Links_Frei}
 {Rechts_Frei}

 {Hat_Vorrat}

 Exemples:

   IF {Vorne_Frei} THEN {Vor};

   WHILE {Hat_Vorrat} DO {Gib_Ab};

.topic If
 █ IF THEN ELSE
 ███████████████

 IF {Capteur:Sensoren} THEN {Commande:Befehle};

 IF {Capteur:Sensoren} THEN {Commande:Befehle} ELSE {Commande:Befehle};

 IF {Capteur:Sensoren} THEN
 BEGIN
   {Commande:Befehle};
   {Commande:Befehle};
 END;

Si le capteur entre IF et THEN a la valeur TRUE, la commande ou le bloc après
THEN est exécuté. S'il existe également une branche ELSE, celle-ci est exécutée
si la valeur est FALSE.

 Exemple:

   IF {Hat_Vorrat} THEN {Gib_Ab} ELSE {Vor};


Au lieu du capteur, vous pouvez également utiliser une {combinaison logique:Logik}
de plusieurs capteurs.

 Exemple:

   IF Hat_Vorrat AND Vorne_Frei THEN
   BEGIN
     Gib_Ab;
     Vor;
   END;

.topic While
 █ WHILE DO
 ███████████

 WHILE {Capteur:Sensoren} DO {Commande:Befehle};

 WHILE {Capteur:Sensoren} DO
 BEGIN
   {Commande:Befehle};
   {Commande:Befehle};
 END;

Tant que le capteur entre WHILE et DO a la valeur TRUE, la commande ou le bloc
après DO est exécuté.

 Exemple:

   WHILE {Platz_Belegt} DO {Nimm_Auf};

Le capteur est d'abord vérifié pour voir s'il est TRUE, puis seulement la
commande est exécutée. En revanche, {REPEAT UNTIL:Repeat} exécute d'abord la
commande puis vérifie.

Si plusieurs commandes dépendant de WHILE DO doivent être exécutées, un bloc
BEGIN-END doit être utilisé;

 Exemple:

   WHILE Hat_Vorrat DO
   BEGIN
     Gib_Ab;
     Vor;
   END;


Au lieu du capteur, vous pouvez également utiliser une {combinaison logique:Logik}
de plusieurs capteurs.

 Exemple:

   WHILE Hat_Vorrat AND Vorne_Frei DO
   BEGIN
     Gib_Ab;
     Vor;
   END;

.topic Repeat
 █ REPEAT UNTIL
 ███████████████

 REPEAT {Commande:Befehle}; UNTIL {Capteur:Sensoren};

 REPEAT
   {Commande:Befehle};
   {Commande:Befehle};
 UNTIL {Capteur:Sensoren};

La commande ou le bloc entre REPEAT et UNTIL est exécuté jusqu'à ce que le
capteur après UNTIL renvoie la valeur TRUE.

 Exemple:

   REPEAT {Vor}; UNTIL NOT {Vorne_Frei};


{REPEAT UNTIL:Repeat} exécute d'abord la commande puis vérifie si le capteur est
TRUE. En revanche, avec {WHILE DO:While}, le capteur est d'abord vérifié pour
voir s'il est TRUE, puis seulement la commande est exécutée.

Au lieu du capteur, vous pouvez également utiliser une {combinaison logique:Logik}
de plusieurs capteurs.

 Exemple:

   REPEAT
     Gib_Ab;
     Vor;
   UNTIL NOT (Hat_Vorrat AND Vorne_Frei);

.topic Logik
 █ Opérateurs logiques
 ██████████████████████

Les {capteurs:Sensoren} peuvent être combinés en PASCAL avec des opérateurs
logiques. Vous pouvez ainsi réaliser des requêtes qui dépendent de plusieurs
capteurs.

 Exemple:

   WHILE (Hat_Vorrat AND Vorne_Frei) DO
   BEGIN
     Gib_Ab;
     Vor;
   END;

Deux opérateurs sont disponibles: OR et AND.


L'opérateur OR combine deux {capteurs:Sensoren} de telle sorte que la valeur de
retour est TRUE si au moins l'un des deux renvoie TRUE.

 Exemple:

   IF Links_Frei OR Rechts_Frei THEN Vor;

 Table de vérité:

   Links_Frei OR Rechts_Frei =
   ████████████████████████████████████
   FALSE      │  FALSE       │ FALSE
   TRUE       │  FALSE       │ TRUE
   FALSE      │  TRUE        │ TRUE
   TRUE       │  TRUE        │ TRUE
   ████████████████████████████████████


L'opérateur AND combine deux {capteurs:Sensoren} de telle sorte que la valeur de
retour est TRUE si les deux {capteurs:Sensoren} renvoient TRUE.

 Exemple:

   IF Links_Frei AND Rechts_Frei THEN Vor;

 Table de vérité:

   Links_Frei AND Rechts_Frei =
   ████████████████████████████████████
   FALSE      │  FALSE       │ FALSE
   TRUE       │  FALSE       │ FALSE
   FALSE      │  TRUE        │ FALSE
   TRUE       │  TRUE        │ TRUE
   ████████████████████████████████████


En utilisant plus d'un opérateur, vous pouvez également combiner plusieurs
capteurs.

 Exemple:

   IF Links_Frei OR Rechts_Frei OR Vorne_Frei THEN Nimm_Auf;

Lorsque différents opérateurs apparaissent dans une telle combinaison, la
question de l'ordre d'évaluation se pose. La règle générale est: NOT avant AND,
AND avant OR. Cela signifie que les combinaisons NOT sont évaluées en premier,
puis AND, et enfin OR.

 Exemple:

   IF Vorne_Frei OR Links_Frei AND Rechts_Frei THEN ...

   équivaut à

   IF Vorne_Frei OR (Links_Frei AND Rechts_Frei) THEN ...


 Priorité       équivalent en mathématiques
 ████████████████████████████████████████████
 1   │  NOT    │ - (négation)
 2   │  AND    │ */ (multiplication/division)
 3   │  OR     │ +- (addition/soustraction)
 ████████████████████████████████████████████

.topic Unterprogramme
 █ Sous-programmes
 ██████████████████

En PASCAL, vous pouvez remplacer des parties identiques d'un programme par des
sous-programmes. Ces sous-programmes peuvent ensuite être utilisés exactement
comme de vraies {commandes:Befehle}.

Ces sous-programmes sont appelés "procédures" en PASCAL. Ils sont définis avant
le programme principal et sont structurés comme de petits programmes PASCAL.

 Exemple:

   PROGRAM Exemple;

     PROCEDURE Drehe_Rechts;
     BEGIN
       Drehe_Links;
       Drehe_Links;
       Drehe_Links;
     END;

   BEGIN
     Vor;
     Drehe_Rechts;
   END.

Entre BEGIN et END, n'importe quelles {commandes:Befehle} et {structures:Strukturen}
peuvent être utilisées. Lorsqu'une procédure est appelée, les commandes de la
procédure sont exécutées les unes après les autres. Ensuite, l'exécution revient
au point d'appel.

Le programme ci-dessus pourrait donc aussi être écrit comme suit:

  PROGRAM Exemple;
  BEGIN
    Vor;
    Drehe_Links;
    Drehe_Links;
    Drehe_Links;
  END;


Cependant, dès qu'un programme doit tourner à droite plusieurs fois, vous devriez
utiliser une procédure pour cela, car le programme devient ainsi plus court et
surtout plus clair. Les procédures font donc partie d'un bon style de
programmation.


Les procédures peuvent s'appeler elles-mêmes ou appeler d'autres procédures, à
condition qu'elles soient déjà définies au moment de l'appel. Elles doivent donc
être définies au-dessus de l'appel dans le texte du programme. Lorsque les
procédures s'appellent elles-mêmes, on appelle cela la "programmation récursive".
Lors de la programmation récursive, il faut veiller à ce qu'aucune boucle infinie
ne se forme, sinon un "dépassement de pile" se produit à un moment donné et le
programme est interrompu. Avec de vrais programmes PASCAL, cela peut même
provoquer un plantage de l'ordinateur.

 Exemple:

   PROGRAM Faux;

     PROCEDURE Gehe;
     BEGIN
       Vor;
       Gehe;
     END;

   BEGIN
     Gehe;
   END.

Comme la plupart des problèmes peuvent également être résolus sans cette
technique parfois déroutante, la programmation linéaire devrait être préférée.
La programmation récursive ne devrait vraiment être utilisée que là où c'est
absolument nécessaire.

.topic Aufbau
 █ Structure des programmes PASCAL
 ██████████████████████████████████

Un programme PASCAL ou programme Niki a toujours la même structure:

La première ligne spécifie un nom pour le programme.

  PROGRAM Nom;

Le nom ne doit pas être identique à un mot-clé PASCAL prédéfini ou à une
commande de Niki. Un programme nommé "Vor" n'est donc pas possible.

Ensuite viennent toutes les définitions. Dans tvNiki, ce sont toutes les
{procédures:Unterprogramme} définies.

Enfin, chaque programme PASCAL doit contenir un programme principal. Ce
programme principal commence par BEGIN et se termine par END suivi d'un point.
Entre les deux, des {commandes:Befehle} peuvent être placées. Chaque commande
se termine par un point-virgule.

  BEGIN
    {Drehe_Links};
    {Vor};
  END.

Au démarrage du programme, l'exécution commence à la première commande du
programme principal.

Les programmes PASCAL valides peuvent donc ressembler à ceci:

  PROGRAM Exemple1;

  BEGIN
    {Vor};
    {Vor};
    {Drehe_Links};
  END.


  PROGRAM Exemple2;

    {PROCEDURE:Unterprogramme} Ranger;
    BEGIN
      {Drehe_Links};
      {Vor};
      {Gib_Ab};
      {Drehe_Links};
      {Drehe_Links};
      {Vor};
      {Drehe_Links};
    END;

  BEGIN
    {Nimm_Auf};
    {Nimm_Auf};
    {Nimm_Auf};
    Ranger;
    Ranger;
    Ranger;
  END.
